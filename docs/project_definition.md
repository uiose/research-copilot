# 项目定义
> 持续更新。最近更新：2026-04-14

## 1、项目目标

**长期目标：**
做一个面向初级研究者的 Research Copilot，能够根据用户提出的研究问题检索相关文献，并生成结构化的文献调研结果。

**项目采用双轨并行策略：**

| 轨道 | 内容 | 每周时间占比 |
|---|---|---|
| **工程轨** | 按路线图推进 Research Copilot 代码 | 60% |
| **研究轨** | 读论文、写 paper_memo、定研究问题 | 40% |

---

**当前目标：**

Week 01（✅ 基本完成）：
- 完成最小原型 baseline，跑通结构化问答，接入大语言模型
- 补 `docs/paper_memo.md`（问题定义 + 论文阅读记录）

Week 02（即将开始）：
- 引入 LangGraph 最小图
- 接入 arXiv / Semantic Scholar 检索工具
- 完成 A/B 对比实验（有工具 vs 无工具）

## 2、用户是谁
用户是初级研究者、研究生，或者刚进入某个方向、需要快速了解领域基本情况的人。
典型特征：有基本学术背景，但缺乏文献调研经验；可能英文阅读能力有限；缺少导师的密切指导。

## 3、输入是什么
输入是一个研究主题或研究问题，通常表现为用户对某个领域、方法或方向的提问。

## 4、输出是什么
当前 baseline 版本的输出为结构化 JSON，包括：
- `topic`：当前处理的研究主题
- `assumptions`：当前版本的假设、边界和限制
- `next_step`：下一步建议动作或后续迭代方向

目标输出（Week 6+）：
- 结构化的文献调研报告，包含：相关论文列表、按主题分组的摘要、研究空白识别、建议的研究方向

## 5、成功标准
- 程序可以运行
- 能接收一个研究主题作为输入
- 输出是固定格式的结构化 JSON
- 已建立固定评测题集
- 程序可以读取并检查题集字段
- `docs/paper_memo.md` 包含问题定义和论文阅读记录

## 7、12 周路线图概览

| 阶段 | 周 | 工程轨重点 | 研究轨重点 |
|---|---|---|---|
| Phase 1 基线建立 | W1-2 | 最小骨架 + LangGraph + 检索工具 | 问题定义 + 读 5 篇论文 |
| Phase 2 能力扩展 | W3-6 | 文献库 + 记忆 + 规划 + 综述生成 | 读 15-20 篇 + 定研究问题 |
